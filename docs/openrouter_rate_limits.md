# OpenRouter API 并发限制

> 记录时间：2026-03-27

## 核心规则：$1 余额 = 1 RPS

并发上限与账户余额直接挂钩，动态调整：

| 余额 | 并发上限 (RPS) |
|------|-------------|
| $1 | 1 |
| $5 | 5 |
| $10 | 10 |
| $50 | 50 |
| $500+ | 500 (上限，更高需联系客服) |

余额消耗后 RPS 同步下降。

## 免费模型限制 (model_id 以 `:free` 结尾)

- 20 requests/min
- 充值 < $10: 50 次/天
- 充值 >= $10: 1000 次/天

## 对 FPE 的影响

| 场景 | 建议并发 (`-c`) | 最低余额 |
|------|---------------|---------|
| 单文件 < 10 页 | `-c 5` | $5 |
| 单文件 10-50 页 | `-c 10` | $10 |
| 批量目录 | `-c 3` (保守) | $5 |
| 高并发压测 | `-c 50` | $50 |
